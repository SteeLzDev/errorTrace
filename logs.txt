Para o controle de envio ficou acordado que serão criados 2 registros na tb_dados_consignante: uma para registrar o mês do envio de SMS e outro para registrar a quantidade de envio no mês.
Quando for feito um envio de SMS, deve verificar se para o mês atual existe o registro com a data de envio.
Se não houver, criar o registro de data e também registrar a quantidade de envio.
Se já houver no mesmo mês, apenas incrementar a quantidade de envio.
Se já houver mas for em outro mês, atualizar a data para o mês atual e reiniciar a contagem da quantidade de envio.
O registo de quantidade de envio e data devem ser utilizados para comparar com os parâmetros de limite inferior e superior de envio de SMS.
Em todos os casos deve-se registrar ocorrência de usuário (tb_ocorrencia_usuario) para cada envio de SMS.
